# v1.3.0 Electron 自动化测试设计

## 目标

为 `notebook-electron` 建立独立于 Spring/Java 版的自动化测试入口，覆盖本地桌面端最核心、
且不依赖真实窗口或外部 AI 服务的业务模块。

## 技术方案

- 使用 Node.js 内置的 `node:test` 和 `node:assert/strict`，不新增测试框架依赖。
- PowerShell 统一运行命令：`npm test`。
- 测试并发数固定为 1，避免共享模块状态和临时资源相互干扰。
- SQLite 测试使用 `:memory:` 内存数据库，不读取或修改用户真实数据库。
- 向量库测试使用系统临时目录中的 JSON 文件，测试结束后自动清理。
- 不读取真实 API Key，不调用 DeepSeek、智谱或 LlamaParse。

## 首批覆盖范围

### 1. 文本提取清洗

- 控制字符清理，同时保留换行与制表符。
- 重复页眉/页脚过滤。
- 疑似水印行过滤。
- 连续空行规整。
- 不支持的文件格式应明确报错。

### 2. 本地向量库

- 新增、同 ID 更新与 JSON 持久化。
- 余弦相似度排序、元数据过滤及维度不匹配处理。
- 中英文关键词检索。
- 按块 ID、按文档 ID 删除并持久化。
- 重新加载文件后数据保持一致。

### 3. SQLite 数据层

- 笔记本与文档 CRUD。
- 文档摘要、分块数更新。
- 删除文档时级联删除对应聊天记录。
- 删除笔记本时级联删除所属文档和聊天记录。
- 对话历史顺序、会话隔离及超长截断。

## 暂不覆盖

- Electron BrowserWindow 和真实 IPC 点击流程。
- 真实文件选择窗口。
- 外部 AI、Embedding 和 LlamaParse 网络调用。

这些场景后续应分别使用 Electron 进程测试或显式启用的联网测试，避免普通 `npm test`
误开窗口、消耗 API 额度或受网络波动影响。
