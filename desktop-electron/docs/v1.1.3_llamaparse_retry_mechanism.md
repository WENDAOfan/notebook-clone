# v1.1.3 - 引入智能网络重载：支持网络抖动自动重试 (Retry Mechanism)

## 问题现象与原因分析

在 Electron 主进程运行中，用户遇到以下报错：
```
[LlamaParse] 上传成功，Job ID: 5579a4f0-a9d7-4b28-b0ff-17bd2a2f7207，开始轮询...
[LlamaParse] 轮询 1/60，状态: SUCCESS
[Extractor] LlamaParse 解析失败，降级至本地（LiteParse/mammoth）。原因: fetch failed Error: Client network socket disconnected before secure TLS connection was established
    at connResetException (node:internal/errors:720:14)
    at TLSSocket.onConnectEnd (node:_tls_wrap:1658:19)
    ...
  code: 'ECONNRESET',
  host: 'api.cloud.llamaindex.ai',
  port: 443
}
```

### 根源分析
1. **成功轮询后的连接瞬间关闭与重建**：
   在第二步（轮询）检测到 `SUCCESS` 后，应用立即（无延迟）发起了第三步（下载 Markdown）的 HTTP 请求。由于我们在 `v1.1.1` 中加入了 `Connection: close` 头，轮询连接在结束后被物理关闭，使得第三步必须建立一个**全新的 TCP 连接并进行完整的 TLS 安全握手**。
2. **TLS 握手被重置 (ECONNRESET)**：
   在短时间内连续向同一个海外主机域名快速建立并关闭多个 TLS 连接，极易触发以下问题之一：
   - 远程服务器（Cloudflare/CDN）的**抗 DDoS 频次限制机制**，直接发送 RST 包拒绝第二次握手。
   - 本地网络防火墙或代理网关检测到高频的 TLS 握手特征，进行了防火墙连接阻断。
   最终在 TLS 握手尚未完成前，客户端 socket 遭到断开，抛出 `Client network socket disconnected before secure TLS connection was established`。

---

## 解决方案设计

网络波动和服务器频次限制是云端 API RAG 管道中普遍存在的现象。解决此类非持久性网络故障（Transient Faults）的最佳工程实践是引入**自动指数退避/重试机制 (Retry with Backoff)**。

### 1. 实现 `fetchWithRetry` 包装器
在 `extractor.js` 中编写一个通用的带有重试特性的 `fetch` 辅助函数：
```javascript
/**
 * 带有自动重试机制的 fetch 封装器
 * @param {string} url 请求地址
 * @param {object} options fetch 参数项
 * @param {number} maxRetries 最大重试次数，默认 3 次
 * @param {number} delay 每次重试前的等待时间（毫秒），默认 1500 毫秒
 */
async function fetchWithRetry(url, options = {}, maxRetries = 3, delay = 1500) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      return await fetch(url, options);
    } catch (err) {
      const isRetryable = err.name === 'TypeError' 
        || err.code === 'ECONNRESET' 
        || err.message.includes('fetch failed');

      if (isRetryable && i < maxRetries - 1) {
        console.warn(`[Network] 请求失败 (${err.message})。正在进行第 ${i + 1}/${maxRetries - 1} 次自动重试...`);
        await sleep(delay);
        continue;
      }
      throw err;
    }
  }
}
```

### 2. 优化连接策略
- 允许继续使用 Keep-Alive（将 `Connection: close` 从 headers 中移除，由 `undici` 全局连接池自行复用 socket），因为即使发生 Keep-Alive 导致的 `ECONNRESET`（Socket Hang Up），重试机制也会自动重新开启新 socket 并重新请求，性能更好。
- 将 `extractor.js` 中所有对 LlamaParse API 的 `fetch` 调用替换为 `fetchWithRetry`。
