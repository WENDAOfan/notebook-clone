# v1.0.1 - 修复流式问答输出时的 Markdown 渲染乱码问题

## 问题背景
用户反馈在生成回答的过程中，内容会显示很多“奇怪的符号”。经过审查 `notebook-electron/renderer/js/app.js` 代码，发现问题出在流式返回（Streaming）期间的 DOM 渲染逻辑。

在原来的实现中：
```javascript
(chunk) => {
    ai.rawText += chunk;
    ai.textEl.textContent = ai.rawText; // 直接插入纯文本
}
```
这会导致用户在生成期间直接看到带有 `**bold**`, `# heading`, `---`, 以及引用格式如 `[1]` 的纯 Markdown 文本，使得视觉呈现像乱码和奇怪的符号。

## 解决办法
我们将接收到的流式数据，实时经过 `marked.parse` 转换为 HTML，并通过 `DOMPurify.sanitize` 清洗，最后使用 `innerHTML` 插入：

```javascript
(chunk) => {
    ai.rawText += chunk;
    const { answer } = parseCitations(ai.rawText, currentDocTitle); // 分离正文与可能在流中出现的引用部分
    ai.textEl.innerHTML = DOMPurify.sanitize(marked.parse(answer)); // 实时渲染Markdown并进行安全过滤
    chatHistory.scrollTop = chatHistory.scrollHeight;
}
```

这确保了即使在缓慢生成的情况下，用户依然能直接看到格式化后干净优美的标题、加粗文本和段落排版，不再显示 Markdown 的原生标记符号。
